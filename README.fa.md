# WhiteDNS - راهنمای فارسی

WhiteDNS یک برنامه اندروید برای اجرای کلاینت تونل DNS به صورت محلی است. برنامه
دو حالت اصلی دارد:

- حالت Proxy برای استفاده با برنامه‌هایی که می‌توانند به یک پروکسی محلی وصل شوند.
- حالت VPN برای عبور دادن ترافیک برنامه‌های اندروید از داخل تونل WhiteDNS.

این فایل نقطه شروع مستندات فارسی پروژه است. برای استفاده روزمره، ابتدا
[راهنمای کاربر](docs/fa/WHITE_DNS_USER_GUIDE.md) را بخوانید. برای تست و انتشار،
[برنامه QA فارسی](docs/fa/WHITE_DNS_QA.md) را ببینید. برای خطایابی، از
[راهنمای عیب‌یابی](docs/fa/WHITE_DNS_TROUBLESHOOTING.md) استفاده کنید.

> **هشدار مهم:** WhiteDNS در Google Play منتشر نشده است. هر APK یا صفحه‌ای که در
> Google Play یا فروشگاه‌های متفرقه با نام WhiteDNS دیده می‌شود، انتشار رسمی این
> پروژه نیست و ممکن است تغییر یافته، قدیمی یا ناامن باشد.

کانال رسمی: [https://t.me/whitedns](https://t.me/whitedns)

## WhiteDNS چه کاری انجام می‌دهد؟

WhiteDNS ترافیک شبکه را مستقیما رمزگشایی یا فیلتر نمی‌کند. نقش برنامه این است که
یک مسیر محلی برای تونل DNS بسازد:

1. برنامه یک پروفایل اتصال شامل دامنه سرور و کلید رمزنگاری می‌گیرد.
2. یک فهرست Resolver برای ارسال درخواست‌های DNS استفاده می‌شود.
3. StormDNS به عنوان موتور تونل اجرا می‌شود.
4. خروجی تونل یا به صورت پروکسی محلی در اختیار برنامه‌های دیگر قرار می‌گیرد، یا
   از طریق VPN اندروید برای برنامه‌های انتخاب‌شده استفاده می‌شود.

اگر برنامه وضعیت «متصل» نشان دهد ولی سایت‌ها یا برنامه‌ها باز نشوند، مشکل می‌تواند
در یکی از این لایه‌ها باشد: سرور، Resolverها، مسیر پروکسی/VPN، تنظیمات شبکه،
MTU، IPv6، یا محدودیت‌های باتری اندروید.

## مسیر پیشنهادی برای شروع

1. یک پروفایل اتصال بسازید.
   - دامنه یا دامنه‌های سرور را وارد کنید.
   - کلید رمزنگاری را وارد کنید.
   - اگر چند دامنه دارید، هر دامنه باید معتبر و مربوط به همان سرور باشد.

2. یک پروفایل Resolver بسازید.
   - با یک فهرست تمیز و کوچک شروع کنید.
   - فهرست‌های خیلی بزرگ، تکراری یا شامل IPهای خصوصی می‌توانند اتصال را کند یا
     ناپایدار کنند.

3. حالت اجرا را انتخاب کنید.
   - اگر می‌خواهید برنامه‌ای مثل کلاینت‌های پروکسی از آدرس محلی استفاده کند،
     حالت Proxy را انتخاب کنید.
   - اگر می‌خواهید ترافیک برنامه‌های اندروید از طریق WhiteDNS عبور کند، حالت VPN
     را انتخاب کنید.

4. اتصال را شروع کنید و وضعیت را مرحله به مرحله بخوانید.
   - اجرای StormDNS به تنهایی کافی نیست.
   - Resolverها باید پاسخ مفید بدهند.
   - مسیر محلی Proxy یا VPN هم باید تست خروجی را پاس کند.

## مفاهیم اصلی

### پروفایل اتصال

پروفایل اتصال مشخص می‌کند WhiteDNS به کدام سرور وصل شود و با چه کلیدی با آن صحبت
کند. دامنه اشتباه یا کلید اشتباه ممکن است باعث شود برنامه تلاش برای اتصال را نشان
دهد، اما ترافیک واقعی کار نکند.

### Resolver

Resolverها مسیر حمل بسته‌های DNS هستند. تعداد زیاد Resolver همیشه بهتر نیست.
Resolver خراب، کند، تکراری، خصوصی یا فیلترشده می‌تواند باعث این مشکلات شود:

- گیر کردن در شروع اتصال.
- مصرف آپلود بیشتر.
- وصل شدن ظاهری ولی باز نشدن برنامه‌ها.
- قطع و وصل شدن‌های پشت سر هم.

### حالت Proxy

در حالت Proxy، برنامه یک پروکسی محلی می‌سازد. اگر آدرس Listen روی Loopback باشد،
فقط خود دستگاه می‌تواند از آن استفاده کند. اگر روی آدرس قابل دسترسی از شبکه محلی
باشد، باید احراز هویت SOCKS با نام کاربری و رمز عبور غیرخالی فعال باشد.

### حالت VPN

در حالت VPN، WhiteDNS از `VpnService` اندروید استفاده می‌کند. این حالت برای کاربر
ساده‌تر است، اما به مجوز VPN، اعلان Foreground Service، تنظیمات باتری، تغییر شبکه،
Split Tunnel، MTU و IPv6 حساس‌تر است.

## امنیت و اشتراک‌گذاری

- لینک پروفایل شامل اطلاعات محرمانه اتصال است.
- QR پروفایل هم همان اطلاعات محرمانه را دارد.
- فقط پروفایل را با دستگاه‌ها یا افراد مورد اعتماد به اشتراک بگذارید.
- خروجی Diagnostics باید به صورت پیش‌فرض Redacted باشد و کلیدها، رمزها، لینک
  پروفایل، مسیر فایل‌های Runtime و محتوای حساس را نشان ندهد.
- فایل‌های Resolver یا پروفایل دریافتی از دیگران را ورودی غیرقابل اعتماد فرض کنید.
