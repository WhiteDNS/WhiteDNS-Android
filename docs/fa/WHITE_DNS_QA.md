# برنامه QA فارسی WhiteDNS

این سند برای تست دستی و نیمه‌دستی WhiteDNS قبل از انتشار یا قبل از Merge تغییرات
مهم استفاده می‌شود. هدف این است که رفتار برنامه از دید کاربر واقعی بررسی شود:
اتصال، امنیت، Diagnostics، UI، مصرف ترافیک، باتری، وارد کردن پروفایل، Resolverها،
Proxy، VPN و راهنمای داخل برنامه.

## معیارهای شدت

| شدت | معنی |
| --- | --- |
| P0 | مانع انتشار: نشت Secret، Crash، اتصال خراب، Export/Import خراب، Exposure ناامن در LAN. |
| P1 | باید قبل از انتشار پایدار حل شود: وضعیت گمراه‌کننده، قطع پس‌زمینه، UI شکسته، مصرف ترافیک غیرمنتظره. |
| P2 | بهتر است زود حل شود: متن مبهم، راهنمای ناقص، کندی UI، خطای Edge Case. |
| P3 | بهبود کم‌ریسک یا ظاهری. |

## محیط‌های تست

- نصب تازه روی نسخه جدید Android.
- Upgrade از نسخه عمومی قبلی.
- یک دستگاه قدیمی‌تر در محدوده پشتیبانی.
- Wi-Fi.
- Mobile Data.
- Battery Saver فعال.
- حالت محدودیت پس‌زمینه.
- دستگاه با تعداد زیاد برنامه نصب‌شده برای Split Tunnel.
- اندازه فونت بزرگ و خیلی بزرگ.

## داده‌های تست

- یک پروفایل سرور سالم.
- یک پروفایل با دامنه خراب.
- یک پروفایل با کلید اشتباه.
- یک پروفایل تک‌دامنه.
- یک پروفایل چنددامنه.
- یک فایل Resolver کوچک و سالم.
- یک فایل Resolver بزرگ.
- یک فایل Resolver با Duplicate.
- یک فایل Resolver با IP خصوصی و Loopback.
- یک فایل Resolver با ردیف‌های نامعتبر.
- نام‌های طولانی برای پروفایل و Resolver.

## تست راهنمای خطای Route

### QA-FA-HEALTH-001 پنل What To Check Next

اولویت: P1

مراحل:

1. پروفایلی تنظیم کنید که Runtime شروع شود ولی Route Check شکست بخورد.
2. همین حالت را در Proxy Mode تست کنید.
3. پنل راهنمای بعدی را زیر وضعیت Verification بررسی کنید.
4. همین تست را در VPN Mode تکرار کنید.

نتیجه مورد انتظار:

- برنامه بعد از شکست Route Check فقط وضعیت مبهم نشان ندهد.
- پنل باید ترتیب سرور، Resolver، مسیر محلی و شبکه را پیشنهاد کند.
- در Proxy Mode باید به آدرس محلی و تفاوت پورت SOCKS و HTTP اشاره کند.
- در VPN Mode باید به مجوز VPN، Split Tunnel، IPv6، MTU و محدودیت باتری اشاره کند.
- پنل نباید Secret، لینک پروفایل، مسیر فایل یا لاگ خام نشان دهد.

## چک‌لیست انتشار

- نصب تازه سالم است.
- Upgrade سالم است.
- Export و QR پروفایل قابل Import هستند.
- Diagnostics به صورت پیش‌فرض Redacted است.
- Proxy Loopback کار می‌کند.
- LAN Proxy بدون Auth کامل Block می‌شود.
- VPN وصل و تمیز قطع می‌شود.
- Recovery بعد از تغییر شبکه محدود و Same-session است.
- Connected-but-no-traffic به عنوان Ready نمایش داده نمی‌شود.

- باینری‌های Native با نسخه و Hash مستند همخوانی دارند.
